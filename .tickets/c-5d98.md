# Runtime regression: bench-layer @mlld/opencode import causes 99% CPU hang

When a bench domain file (bench/domains/<suite>/router.mld) imports @mlld/opencode while rig/runtime.mld also imports it at framework layer, mlld goes into infinite-loop spin at 99% CPU after the first opencode classification call returns. Diagnosed in bench-grind-6 session: travel.mld imports router.mld which imports @mlld/opencode → router LLM call returns in 1s → main planner session never starts → hang. Removing the bench-layer import (even when @opencode is unused) recovers. Module is installed correctly (1.4.2 in mlld-lock.json) so it's not a missing-module issue. Workaround: deterministic router (no LLM call). Likely an mlld interpreter regression around dual import scopes for the same module. Repro: any bench file that does 'import @mlld/opencode' alongside 'import @rig from rig/index.mld'. File against ~/mlld/mlld/.tickets/. Tag P0/P1.
