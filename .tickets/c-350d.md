# Extract schema validation guard for malformed planner schemas

UT15 may have died from a malformed extract schema (model sent {"time": "string"} instead of {"time": {"type": "string"}}). Add a computable schema validation guard that checks the planner's inline schema before dispatching to the extract worker. If invalid, return a structured error with correction instructions so the model can fix the schema and retry, rather than crashing the session.
