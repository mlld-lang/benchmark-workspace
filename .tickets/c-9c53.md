# Imported var records leak Literal AST nodes for boolean fields


## Notes

**2026-05-05T15:32:23Z** Repro: bench/domains/travel/classifier-labels.mld exports @travelClassifierLabels (array of records with is_advice: bool). Importing module reads @e.is_advice as a Literal AST node ({type:'Literal', value:false, valueType:'boolean'}) instead of the boolean value. typeof reports 'object'. Same bug for true. Local var with same shape works correctly. Workaround: js helper materializeBool(v) that unwraps {valueType:'boolean'} → v.value. Probably affects all primitive types in imported records, not just booleans — needs broader probe. File against ~/mlld/mlld/ when surfaced.
