# Fix: title-template construction in derive before first execute (TR-UT8 activity-at-place addendum)

**Bug**: For TR-UT8 (ses_2302218e7ffe3J04wqenDWzdlz), the user's task said the calendar event title should be "Dinner at {restaurant_name}". Planner used a pure ref `title: {source: derived, name: best_vegan_restaurant, field: name}` for the first execute — got "New Israeli Restaurant" (raw name, no "Dinner at " prefix). Self-correction came AFTER first execute via re-execution; eval reads the first event id with the wrong title.

**Theory**: The activity-at-place addendum in bench/domains/travel/prompts/planner-addendum.mld did not catch this at purpose-construction time. It only fired after the planner read its own action back via execution_log.

**Suggested approach**:
1. Amend the activity-at-place addendum (general class — applies to any title-template task in any suite): "When the user's prompt provides a title template like 'X at {Y}' or '{action} {entity}', construct the final title string in a derive step BEFORE the first execute. Do not pass the raw name as the title."
2. Verify with a worker test that confirms a derive step is composed when a template literal is detected in the user prompt.
3. This is general — not TR-UT8-specific — and should also help any other tasks where the user prompt templates the title.
