# Compose @wrapTools lifecycle helper for rig (planner-wrapper consolidation)

User-space composition helper that wraps a tool collection with before/after lifecycle behavior using mlld's existing primitives (guard before/after tool:* + session-scoped state). Goal: collapse the six near-identical planner wrappers in `clean/rig/workers/planner.mld:261+` into one declarative wrap.

Sketch signature:
```
var @plannerTools = @wrapTools(@baseTools, {
  before: @recordBudget,
  after: @appendExecutionLog,
  budget: 20,
  onError: @terminateConversation
})
```

Implementation surface:
- Pure mlld composition. No interpreter changes.
- Uses `guard before tool:*` / `guard after tool:*` (already shipped) for hooks.
- Uses session-scoped state (already shipped) for budget counting and execution-log appending.
- Probably ~30 lines of mlld in `clean/rig/`.

## Dependency on c-1f50: preference, not technical

The original ticket said "defer until c-1f50 lands." That's a soft preference, not a hard block. The `@wrapTools` sketch above does not reference any `.mx.*` accessor; it composes guards and session state, which are independent of how rig reads tool metadata. If c-1f50 stays partial or stalls, c-775b can still proceed — the worst case is that one or two leaf reads inside `@wrapTools` go through `@tool*` helpers instead of `.mx.*`, and a follow-up cleans those up later.

The "migration may reveal the right shape" framing in the original body is fine as a preference, but it shouldn't prevent us from doing this work if it becomes the higher-priority cleanup. Decouple if it does.

## Acceptance criteria

- Helper defined in `clean/rig/` (likely `lifecycle.mld` or a new file).
- The six planner-tool wrappers in `workers/planner.mld:261+` collapse to one composition.
- `clean/rig/tests/index.mld --no-checkpoint` passes.
- No new mlld interpreter changes needed.
- A targeted bench sweep on one suite shows no regression vs the prior numbers in STATUS.md.

## Why this is worth doing

The six wrappers are real duplication and any change to lifecycle behavior (budget accounting, log shape, error handling) currently requires editing six places. One declarative wrap removes that maintenance tax.

## What this is NOT

- Not a primitive request — it's pure mlld composition over already-shipped primitives.
- Not a performance fix.
- Not a fan-out of new behavior — same lifecycle semantics, just expressed once.
