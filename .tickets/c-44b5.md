# Revisit resolved-value passing to inner workers

The derive worker needs resolved handles to construct selection refs, but currently only sees raw values. We added a handleMap parameter as a workaround. The shelf/session mechanism was intended to solve this kind of state passing (shelf_notes auto-injection), but the v2 architecture may need a different approach. Investigate: (1) whether var session could carry resolved handles to inner workers automatically, (2) whether the display projection system could annotate values with their handles, (3) whether the current handleMap approach is actually fine and just needs polish. The question is about ergonomics and conventions, not just making it work.
