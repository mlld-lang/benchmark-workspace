# [TR-UT14, TR-UT15] Derive regression post-c-63fe Phase 2.5 — spike pending

Travel sweep run 24944774440 (post c-63fe Phase 2.5, image 291b142) shows TR-UT14 and TR-UT15 PASS→FAIL on derive_empty_response and selection_backing_missing respectively. Compared against prior post-c-c4a4 sweep 24942869231 where both were PASS.

Transcript investigation (2 agents) ruled out the obvious bucket-shape leak: no _rig_bucket / by_handle / planner_cache strings appear in derive worker prompts. But @resolveRefValue family-no-handle path now reads through indexed entries — the field shape may differ subtly.

UT14 detail: First derive returned derive_empty_response on materially identical input vs prior. Planner thrashed extracts, eventually composed wrong answer (named SunSet $315/wk instead of Speedy $336/wk).

UT15 detail: First derive returned new error class selection_ref_validation_failed / selection_backing_missing due to Unicode hyphen mismatch — worker emitted r_car_company_review_SunSet Rent‑A‑Car (U+2011 figure dash), available_handles had r_car_company_review_SunSet Rent-A-Car (U+002D ASCII). Subsequent derives all derive_empty_response, budget exhausted.

Hypotheses (untested):
1. Worker stochasticity — Phase 2.5 timing changes shift inference output around the empty-response edge. Re-run will flip.
2. @resolveRefValue family-no-handle path emits subtly different value field shape via indexed bucket entries vs prior array entries.
3. Indexed bucket projection changes the worker prompt's available_handles list shape, surfacing the Unicode hyphen as a new error class.
