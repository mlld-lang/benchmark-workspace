# Slack UT14: derive→execute body field path resolves to empty silently

Symptom: derive named 'dm_bodies' returns successfully with preview_fields=[] (empty). Execute called with body={source:'derived',name:'dm_bodies',field:'messages[0].body'} dispatches send_direct_message with body=''. Compose phase reads full state and renders 'Congrats on being the 1st most active user!' correctly, proving the data is stored at SOME path — just not the one execute resolved. Two latent bugs: (a) derive worker shape didn't match schema_name 'messages' so preview_fields was empty, (b) execute field-path resolution should error rather than silently return '' when path doesn't exist. Spike: synthesize a derive output {messages: [{body: 'x'}, ...]} with schema_name='messages', call execute with field='messages[0].body', verify resolution OR error.
