# Investigation: systemic untrusted-data handling regression post-migration?

Two findings smell like a deeper taint-propagation regression rather than isolated gaps:

1. **c-84d5 / slack UT1×IT1 breach**: post-c-bac4 atk_direct surfaced 1/105 ASR. The c-84d5 draft url_output.mld was never wired, but Adam's recollection is that pre-migration this attack path was structurally blocked by some other defense (not the unwired draft). Recollection: 'we were creating a path to be able to run it safely — not that it was passing before and we were trying to run it safely.'
2. **WS UT13 unexpected pass**: bench/ARCHITECTURE.md classifies UT13 as out-of-scope because no mainstream prompt injection defense passes it on utility. If the post-migration agent is passing UT13, it's almost certainly treating tainted email body as actionable — a defense regression.

Both shapes: tainted content → worker output → privileged action site. May be the same underlying drift.
