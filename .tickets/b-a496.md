# mlld SDK hang on classifier output flow in bench context


## Notes

**2026-05-05T19:19:12Z** Repro: bench/agents/travel.mld using @rig.classify with @travelToolRouter + @travelAdviceClassifier (LLM-driven classifiers via Cerebras gpt-oss-120b through @rig.llmCall opencode harness) hangs at 99% CPU, ~2 GB memory, after the classifier fires. macOS 'sample' shows V8 stuck recursively in JsonStringifier::Serialize_ following an fs.FileHandle.ClosePromise — looks like a serialization explosion on a deeply-nested object after an async file close.

The LLM classifier path is parked at bench/domains/travel/classifiers/{advice,tool-router}.mld with passing zero-LLM mlld tests (tests/suites/rig/classify.mld) and live tests (bench/tests/travel-advice-gate-live.mld).

Investigation paths: (a) is the SDK serializing the agent's intermediate values (e.g. classifier results) for tracing/logging that explodes on metadata depth; (b) is mlld's internal cache writing the classifier exe's response (which has @rig.llmCall + record coercion metadata) and that write triggers a recursive stringify; (c) is something specific to import { @tools } from MCP-backed tools.mld interacting with classifier exes from a sibling module.
