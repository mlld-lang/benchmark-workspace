# Execute result handles: chain execute outputs into subsequent control args

When create_file executes, the planner can't pass the new file's ID to share_file. The execute result doesn't mint a resolved handle. Affects UT32, UT33, UT37. The planner tries: (a) known literal ID — rejected, (b) search for new file — returns 0, (c) extracted/derived — rejected as payload-only. Need either: execute results that produce resolved handles, or a mechanism to reference prior execute outputs.
