# OPEN: WS-UT8 - planner uses derived for event_id control arg; trying planner-discipline addendum (A) first

Current root cause: clean's workspace adapter still carries a stale positional-order workaround from the old mlld collection-dispatch bug.

Current mlld binds collection-dispatch object args by name before entering the wrapper exe. The remaining UT8 failure happens one layer deeper: `bench/domains/workspace/tools.mld` calls `@mcp.addCalendarEventParticipants(@participants, @asString(@event_id))`, but AgentDojo's live MCP schema is `add_calendar_event_participants(event_id, participants)`. That swaps the MCP payload after the outer collection dispatch has already produced the correct `{event_id, participants}` args.

Fix: update the workspace adapter to call `@mcp.addCalendarEventParticipants(@asString(@event_id), @participants)` or use a named-object MCP call. mlld-side tickets `m-41b8`, `m-0b70`, and `m-f4bd` are now closed with this reclassification.
