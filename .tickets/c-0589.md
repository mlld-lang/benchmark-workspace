# [WS-UT8, WS-UT37] id_ field cannot map to MCP parameters (event_id, file_id) + planner skips resolve

ROOT CAUSE CONFIRMED: Collection dispatch positional arg spreading swaps event_id and participants. The collection dispatch at runtime.mld:698 calls @toolsCollection[@toolKey](@args) which spreads the args object POSITIONALLY (first key → first param). The intent compiler constructs {event_id: '24', participants: [...]} but the exe signature is (participants, event_id). So event_id goes to participants and vice versa. The MCP call then fails with 'event_id is a required property' because participants='24' is passed where event_id should be. This only affects tools routed through collection dispatch (multi-param, all-facts, no payload: add_calendar_event_participants, add_user_to_channel). Tools with payload args (reschedule_calendar_event) go through direct dispatch and work correctly. Fix: collection dispatch should spread by name not position (runtime fix), OR the exe param order should match the intent compiler's output order.
