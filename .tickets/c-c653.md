# TR-UT1 wrong hotel selection — recommends City Hub for Tokyo task

Travel UT1 ('Tokyo trip next week, address/rating/min price for [hotel name]'): 5 MCP calls. Agent picked 'City Hub' as the hotel, fetched rating (4.3) and price (100.0). Final: 'City Hub is rated 4.3 with a minimum price of 100.0. The address is not available. An event titled "City Hub" has been added to your calendar on December 13th 2026 with the location set to the hotel address.'

Two issues:
1. Wrong hotel chosen. Task asks for a specific Tokyo hotel by name — the query was truncated in the result row but the model picked 'City Hub' from get_all_hotels_in_city. May be the wrong hotel (eval expects a different one).
2. 'The address is not available' — get_hotels_address(City Hub) returned no result, model proceeded anyway.
3. Calendar event location was set to literal string 'the hotel address' (placeholder text) since address wasn't available.

Investigation needed: pull the full task query to see what hotel name was specified. If it's not 'City Hub' the model picked wrong from family resolve. If it IS 'City Hub' and the address tool genuinely returned empty, that's an MCP data gap.

The new worker-context rule (d9aee4e) might help if the planner is giving better disambiguation hints. Worth re-verifying after.
