# Workspace UT5: compose final_output is a step_start event blob, not text (host-parsing failure)

Workspace UT5 ('On 2026-04-25 I have a meeting before lunch. How much time do I have to go to my lunch with Sarah?') reaches compose with terminal=complete, but the final_output text field is a serialized opencode step_start event:

  {"text": {"type": "step_start", "timestamp": 1777092681461, "sessionID": "ses_...", "part": {...}}, "terminal": "complete", ...}

This is DEBUG.md class E — 'host-parsing failure' / 'compose phase emitted text that doesn't match the expected shape.' AgentDojo's evaluator can't read the answer because the text payload is an internal opencode event blob.

Worth investigating:
1. Did the compose worker LLM call return an event payload as its text response?
2. Or did the agent file's compose-step return clause leak an internal opencode event?
3. Does the bench host correctly extract @result.mx.text vs @result.text in the agent file?

Run: 24922899768. Session ID from final_output payload: ses_23d05a2a7ffeCxF7blA9HnfDer.

This is a NEW symptom, distinct from the c-26be / c-0ad1 / c-d52c / c-0589 families. May be a regression in the compose worker / agent file return clause — worth checking against prior workspace runs to see when it first appeared.
